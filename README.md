<p align="center">
  <strong>一个面向 Cursor / VS Code 的本地项目控制台。</strong>
</p>

<p align="center">
  管理文件夹、工作区、项目分组和配置备份，让项目切换更快一点。
</p>

Ask Project Manage 会把你常用的项目目录和 `.code-workspace` 工作区整理到一个可搜索的控制台里。你可以按业务、客户、阶段或团队创建分组，快速打开项目，也可以导出配置，在另一台电脑或另一个编辑器里恢复同一套项目入口。

适合经常在多个仓库、多个工作区、多个 Cursor / VS Code 窗口之间切换的开发者。

## 亮点

- 一处管理文件夹项目和 `.code-workspace` 工作区
- 按分组整理项目，支持搜索名称、路径、类型和分组
- 状态栏一键打开或隐藏项目控制台
- 支持 JSON / YML 导入导出，方便备份和迁移
- 可关闭自动打开面板，也可自定义底部 HUD 统计项
- 首次使用提供新手引导，后续可在设置里再次查看
- 数据只保存在本地编辑器状态中，不上传项目路径和配置

## 快速开始

1. 安装并启用 Ask Project Manage。
2. 打开命令面板，执行 `Project Management: Open Ask Project Manager`。
3. 第一次打开时跟随新手引导了解状态栏入口和设置入口。
4. 点击 `添加分组` 创建你的第一个项目分组。
5. 点击 `导入工作区` 添加 `.code-workspace` 文件。
6. 需要导入普通文件夹时，点击 `导入工作区` 右侧箭头，选择 `导入文件夹`。
7. 后续通过顶部搜索、分组导航或项目卡片快速打开项目。

也可以点击编辑器右下角状态栏里的 `Ask Project Manage`，快速显示或隐藏面板。

## 常用操作

### 导入项目

- `导入工作区`：选择一个或多个 `.code-workspace` 文件。
- `导入文件夹`：选择一个或多个普通项目目录。
- 当前没有分组时，直接导入项目会自动创建默认分组。

### 打开项目

在项目卡片上点击即可打开对应文件夹或工作区。

卡片菜单还支持编辑展示名称、复制路径和删除记录。

删除项目只会从 Ask Project Manage 的列表中移除记录，不会删除磁盘上的真实文件。

### 搜索和分组

顶部搜索框可以按项目名称、路径、类型或分组筛选。

当你有多个分组时，顶部会显示分组导航，可以在 `全部` 和具体分组之间切换。

### 备份和迁移

通过 `导出配置` 可以导出 JSON 或 YML 文件。

换电脑、换编辑器或分享项目入口时，再使用 `导入配置` 恢复即可。

导入配置会和当前列表合并，不会覆盖已有内容。同名分组会自动追加序号，避免误覆盖。

### 偏好设置

右上角设置入口可以调整：

- 是否在编辑器启动后自动打开面板
- 是否显示底部 HUD
- HUD 展示哪些统计项：项目总数、文件夹、工作区、分组
- 是否重新打开新手引导

## 数据存储

Ask Project Manage 使用编辑器扩展的本地全局状态保存数据。

保存内容包括：

- 项目名称
- 项目路径
- 项目类型
- 分组信息
- 偏好设置
- 新手引导已读状态

这些数据不会上传到远端服务。如果需要迁移或备份，请使用 `导出配置`。

## 入口方式

- 命令面板：`Project Management: Open Ask Project Manager`
- 状态栏：点击 `Ask Project Manage`
- 自动打开：默认开启，可在设置中关闭

## 适合这些场景

- 同时维护多个业务项目
- 经常打开不同客户或团队的仓库
- 需要管理多个 `.code-workspace`
- 希望把临时调试项目和长期维护项目分开
- 想替代杂乱的最近打开列表

## 注意事项

- 删除项目不会删除磁盘文件。
- 删除分组会移除该分组下的项目记录。
- 在 `全部` 状态下删除会清空所有分组和项目记录。
- 项目路径不存在时，编辑器会提示路径无效。
- 新手引导完成后不会重复自动弹出，可以在设置里手动再次打开。
