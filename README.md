# Ask Project Manage

<p align="center">
  <img src="dist-webview-app/logo.png" width="128" alt="Ask Project Manage Logo" />
</p>

<p align="center">
  <strong>把 Cursor / VS Code 里的文件夹、工作区和项目分组，整理成一个可搜索、可备份、可迁移的本地项目控制台。</strong>
</p>

Ask Project Manage 是一个用于 Cursor / VS Code 的本地项目管理面板。它把常用的文件夹、`.code-workspace` 工作区和项目分组集中到一个可视化控制台里，让你不必在文件系统、最近打开列表和多个编辑器窗口之间来回翻找项目。

插件首页是一个「灵枢控制台」式的项目台：顶部负责搜索、导入和管理，中间展示项目卡片，底部 HUD 实时汇总项目、文件夹、工作区和分组数量。

## 核心能力

- 统一管理文件夹项目和 `.code-workspace` 工作区
- 按业务、团队、状态或阶段创建项目分组
- 通过项目名称、路径、类型和分组快速搜索
- 支持 JSON / YML 配置导入、导出、备份和迁移
- 支持自动打开面板、底部 HUD、新手引导等偏好设置
- 批量编辑分组和项目名称，调整排序并安全删除
- 所有数据保存在本地编辑器状态中，不上传项目路径和配置

## 适合谁使用

- 经常在多个项目、多个仓库、多个工作区之间切换的开发者
- 需要把项目按业务、团队、客户、环境或阶段分组管理的人
- 希望在 Cursor / VS Code 内直接打开文件夹和 `.code-workspace` 工作区的人
- 希望拥有一个比系统最近打开列表更清晰、更稳定的项目入口的人

## 打开插件

安装并启用扩展后，可以通过以下方式进入项目台：

- 编辑器启动后，扩展默认会自动打开 Ask Project Manage 面板；如果不希望自动弹出，可以在设置中关闭。
- 执行命令 `Project Management: Open Ask Project Manager` 打开面板。
- 点击状态栏里的 `Ask Project Manage`，可以显示或隐藏面板。

首次打开面板时会展示新手引导，说明自动唤起、状态栏快捷入口和再次查看引导的位置。完成引导后，扩展会记录已读状态，后续不会自动弹出；仍然可以在设置中手动再次打开。

## 界面说明

### 顶部操作区

顶部区域包含搜索框、导入按钮和更多操作菜单。

- 搜索框：按项目名称、路径、项目类型或分组名称筛选项目。
- `导入工作区`：默认主按钮，用于导入 `.code-workspace` 文件。
- 导入下拉菜单：展开后可导入工作区、导入文件夹、导入配置，也可以导出配置。
- 右上角更多菜单：用于添加分组、进入批量管理、删除当前分组或删除全部分组。

### 项目列表

中间区域展示项目卡片。每张卡片会显示项目名称、路径、项目类型和所属分组。点击卡片可以打开项目；点击卡片右上角菜单可以执行更多操作。

### 分组导航

当存在多个分组时，顶部会展示分组导航，可以在 `全部` 和具体分组之间切换。当只有一个分组时，导航会自动隐藏，减少不必要的界面干扰。

### 底部 HUD

底部 HUD 展示当前项目台的统计信息：

- 项目总数
- 文件夹数量
- 工作区数量
- 分组数量

这些数据会随着导入、删除、搜索和分组切换实时更新。

如果你希望界面更安静，可以在设置中关闭整个 HUD，也可以只保留部分统计项。

### 设置面板

右上角设置入口用于调整面板行为和展示内容：

- 是否在编辑器启动后自动打开 Ask Project Manage 面板。
- 是否显示底部 HUD。
- 选择 HUD 中显示的统计项：项目总数、文件夹、工作区、分组。
- 重新打开新手引导。

## 快速开始

1. 打开 Ask Project Manage 面板。
2. 第一次使用时，先按新手引导了解自动唤起、状态栏入口和设置入口。
3. 可以先点击 `添加分组` 创建一个分组；也可以直接点击 `导入配置` 恢复已有配置。
4. 点击 `导入工作区` 导入 `.code-workspace` 文件。
5. 需要导入普通项目目录时，点击 `导入工作区` 右侧箭头，在下拉菜单中选择 `导入文件夹`。
6. 项目导入后，可以通过搜索框、分组导航和项目卡片快速定位并打开项目。

## 功能使用说明

### 导入工作区

适合导入 `.code-workspace` 文件。点击顶部主按钮 `导入工作区` 后，在系统文件选择器中选择一个或多个 `.code-workspace` 文件。

导入后，工作区会显示为项目卡片。之后点击卡片即可用编辑器原生能力打开该工作区。

### 导入文件夹

适合导入普通项目目录。点击 `导入工作区` 右侧箭头，选择 `导入文件夹`，然后在系统文件选择器中选择一个或多个文件夹。

导入后，文件夹会显示为项目卡片。之后点击卡片即可在新窗口中打开该目录。

### 自动创建默认分组

如果当前没有任何分组，直接导入文件夹或工作区时，插件会自动创建一个默认分组，并把新项目放入该分组中。

### 添加分组

点击右上角更多菜单，选择 `添加分组`，输入分组名称后确认。

分组适合按自己的工作方式组织项目，例如：

- 按业务线：官网、后台、移动端、AI 工具
- 按团队：前端、服务端、设计协作
- 按状态：常用、归档、临时调试
- 按客户或阶段：交付中、维护中、实验项目

### 删除分组

点击右上角更多菜单中的删除项可以删除分组。

- 当前在具体分组下时，菜单显示 `删除当前分组`，确认后会删除该分组以及分组内所有项目。
- 当前在 `全部` 状态下时，菜单显示 `删除全部分组`，确认后会清空所有分组和所有项目。

删除操作都会弹出确认窗口，避免误删。

### 项目卡片操作

每个项目卡片右上角都有操作菜单：

- `打开项目`：打开该文件夹或工作区。
- `编辑符名`：修改项目在面板中的展示名称，不会修改本地文件夹名或工作区文件名。
- `复制路径`：复制项目路径到剪贴板。
- `删除项目`：从当前分组中移除该项目，删除前会弹出确认窗口。

删除项目只会从 Ask Project Manage 的配置中移除记录，不会删除你磁盘上的真实文件。

### 搜索项目

顶部搜索框支持按以下内容筛选：

- 项目名称
- 项目路径
- 所属分组
- 项目类型

当搜索没有匹配结果时，空状态会提示你清空关键词或导入新的项目。

### 批量管理

点击右上角更多菜单，选择 `批量管理` 进入编辑模式。

在编辑模式中可以集中维护分组和项目：

- 编辑分组名称
- 编辑项目名称
- 调整分组排序
- 调整项目排序
- 删除分组
- 删除项目

删除分组或项目时同样会出现确认窗口。调整完成后点击 `保存` 才会写入配置；点击 `关闭` 会退出编辑模式。

### 导入配置

点击 `导入工作区` 右侧箭头，选择 `导入配置`，然后选择 `.json` 或 `.yml` 配置文件。

导入配置适合这些场景：

- 换电脑后恢复项目台
- 在 Cursor 和 VS Code 之间同步项目列表
- 把同一套项目分组复制给团队成员
- 从备份文件恢复误删的项目列表

导入配置会与当前项目台合并，不会覆盖已有内容。如果导入文件中存在同名分组，插件会自动重命名为 `分组(1)`、`分组(2)` 这类名称，并把导入项目追加到新分组中。

### 导出配置

点击 `导入工作区` 右侧箭头，悬停 `导出配置`，选择导出格式：

- `JSON`
- `YML`

导出的配置文件包含所有分组、项目名称、路径和项目类型。你可以把它作为备份，也可以在其他环境中通过 `导入配置` 恢复。

### 偏好设置

点击右上角设置按钮可以打开偏好设置。

可配置项包括：

- `自动打开面板`：开启后，编辑器启动并激活扩展时会自动打开 Ask Project Manage。
- `显示底部 HUD`：控制首页底部统计面板是否展示。
- `HUD 统计项`：分别控制项目总数、文件夹、工作区、分组是否显示。
- `新手引导`：手动重新打开引导，不会重置已读状态。

## 空状态说明

### 还没有任何分组

当项目台为空时，会展示 `添加分组` 和 `导入配置` 两个入口。

- 新用户可以先添加分组，再导入项目。
- 已有备份配置的用户可以直接导入配置。

### 当前分组没有项目

当某个分组为空时，会展示 `导入文件夹` 和 `导入工作区` 两个入口。导入后的项目会自动收纳到当前分组。

### 搜索没有结果

如果搜索后没有匹配项目，可以尝试清空关键词，或者导入新的文件夹、工作区。

## 配置和数据

Ask Project Manage 的项目列表保存在编辑器扩展的本地全局状态中。它只保存项目路径、项目名称、项目类型和分组信息，不会上传你的项目数据。

如果需要迁移、备份或分享配置，请使用面板中的 `导出配置` 功能。

偏好设置也保存在编辑器扩展的本地全局状态中，包括自动打开面板、HUD 显示项和新手引导已读状态。新手引导只会在 `已读状态` 为未读时自动弹出。

## 常见使用场景

### 管理多个前端项目

把所有前端项目导入到同一个面板中，用分组区分业务线或技术栈。需要切换项目时，打开面板搜索项目名即可进入。

### 管理多根目录工作区

对于同时包含前端、后端、脚本、文档的项目，可以导入 `.code-workspace` 文件，保留原有工作区结构并快速打开。

### 维护临时调试项目

可以创建 `临时` 或 `实验` 分组，把短期排查、验证、迁移用的项目集中放置，避免混入长期维护项目。

### 在多台设备之间迁移项目台

在旧设备上导出 JSON 或 YML 配置，在新设备上导入配置，即可恢复原来的分组和项目入口。

### 替代杂乱的最近打开列表

系统最近列表容易受临时目录干扰。Ask Project Manage 更适合维护一个主动整理过、长期稳定的项目入口。

## 注意事项

- 删除项目只会移除插件中的记录，不会删除磁盘文件。
- 删除分组会同时移除该分组下的项目记录。
- 在 `全部` 状态下删除会清空所有分组和所有项目记录。
- 导入配置会和当前配置合并；同名分组会自动追加序号，避免覆盖已有分组。
- 如果项目路径已经不存在，打开项目时编辑器会提示路径无效。
- 关闭自动打开面板后，仍然可以通过命令面板或状态栏入口手动打开。
- 新手引导完成后不会重复自动弹出，需要时可在设置中再次打开。

## 设计目标

Ask Project Manage 的目标不是替代编辑器原生文件管理，而是补上「项目入口」这一层能力：

- 项目可以被长期保存和整理
- 项目可以被分组、搜索和批量维护
- 文件夹与工作区可以用同一个入口打开
- 项目数量多时依然能快速定位
- 配置可以导入、导出、备份和迁移
- 面板在视觉上更像一个专门为项目切换设计的控制台
